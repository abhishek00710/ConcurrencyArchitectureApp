# Contributing

Thanks for considering a contribution.

This project is most useful when it stays:

- easy to read
- small enough to demo quickly
- realistic enough to teach production-quality concurrency ideas

## Good contribution ideas

- Add another concurrency-focused screen or workflow
- Improve UI polish while keeping the app easy to understand
- Strengthen documentation and teaching notes
- Add tests around cancellation, caching, or task coordination
- Improve accessibility, naming, or code comments

## Before opening a PR

Please try to keep changes aligned with the repository goals:

- prefer clarity over cleverness
- make concurrency boundaries explicit
- keep examples small and explainable
- avoid unnecessary abstraction

## Pull request checklist

- Explain what changed and why it helps learners or users
- Mention any concurrency behavior that changed
- Update `README.md` if the demo flow or screenshots changed
- Add or update tests when behavior changes

## Suggested areas to contribute next

- architecture visualization screen
- performance instrumentation demo
- offline cache demo
- richer test coverage for cancellation races
